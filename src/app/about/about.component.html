Platform Check Page

This tool allows a user to quickly determine if the necessary IDOL components are up and running at any given time. It could be useful after an ISM or any time there has been maintenance done to the environment. All of the environments are displayed in tabular format but we only need to check Production. The necessary components are License Server plus any component like Content Server that is relied upon to serve results to our supported sites.

Each environment/ server for IDOL is displayed in tabs.

Within each tab you will see a list of IDOL components that must be up and running.

To test the environment, click the blue button on the lower left of the screen to check if the License Server and Queryable Ports are available.

A dialog will show the results along with either a check for success or cross for failure.

If a component fails to respond, the service name on that machine is displayed.

Click the button on the lower right side of the screen to see all installed components for IDOL on that particular server. It is NOT necessary to view the details after running a successful check.

Details Page

This page provides detailed information for all installed IDOL components on a given machine.

There are tabs for each IDOL installation on the server. The page acts as documentation of all installations and necessary details for every component installed across all IDOL environments.

It also shows whether the component is active or not.

The Service name: Service name on machine
Executable Path: Executable path on machine button at the top right of the page displays the service name and executable path in a pop-over of the machine's unique License Server that applies to all installs on the machine.

Each installed component* displays either a for components that are up and running or an X for components that are not along with an accordion to display a link to the component's Admin UI and details for each component's executable file path and service name on the machine. This would help troubleshoot any component that is not responding.

*The Find Admin and Data Admin components cannot be queried like the others so they do not display a check or a cross. To verify they are up and running, go to there respective Admin UI links or check their services on the server.
